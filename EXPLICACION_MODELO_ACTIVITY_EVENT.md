## üéØ ¬øQu√© es ActivityEvent?

`ActivityEvent` es el **sistema unificado de logging de actividades** para todo el sistema SGM. Registra **cada acci√≥n importante** que un usuario realiza en el sistema, permitiendo:

1. **Auditor√≠a completa**: ¬øQui√©n hizo qu√© y cu√°ndo?
2. **Timeline de cierres**: Ver toda la historia de un cierre
3. **Debugging**: Rastrear problemas en procesamiento
4. **An√°lisis de uso**: Entender patrones de usuarios

## üîë El Campo Clave: `cierre` (Normalizado)

### **Problema Anterior (V1)**:
```json
{
  "details": {
    "cierre_id": 30,  // ‚ùå Dentro del JSON
    "archivo": "libro.xlsx",
    "filas": 150
  }
}
```

**Problemas**:
- ‚ùå No se puede hacer `WHERE cierre_id = 30` eficientemente
- ‚ùå Queries lentos (full table scan)
- ‚ùå No hay integridad referencial (si eliminas cierre, datos quedan hu√©rfanos)
- ‚ùå No puedes hacer `.filter(cierre=cierre)` directamente

### **Soluci√≥n V2 (Normalizado)**:
```python
ActivityEvent.objects.create(
    cierre=cierre,  # ‚úÖ ForeignKey directo
    details={
        "archivo": "libro.xlsx",  # Solo info contextual
        "filas": 150
    }
)
```

**Beneficios**:
- ‚úÖ Query r√°pido: `ActivityEvent.objects.filter(cierre=cierre)`
- ‚úÖ Index en base de datos: `(cierre_id, timestamp)`
- ‚úÖ Integridad: Si eliminas cierre, elimina eventos autom√°ticamente
- ‚úÖ ORM feliz: `cierre.activityevent_set.all()` funciona directo

```python
@staticmethod
def log(user, cliente, event_type, action, 
        resource_type='general', resource_id='', 
        details=None, session_id='', 
        request=None, cierre=None):
    """
    Registra un evento de actividad.
    
    Args:
        user: Usuario (request.user o User.objects.get())
        cliente: Cliente (cierre.cliente)
        cierre: CierreNomina (NORMALIZADO) ‚úÖ
        event_type: Categor√≠a (upload/process/delete/view)
        action: Acci√≥n espec√≠fica (archivo_validado/procesamiento_iniciado)
        resource_type: Tipo (libro_remuneraciones/movimientos_mes)
        resource_id: ID del recurso (str(libro.id))
        details: Dict con info adicional (NO incluir cierre_id aqu√≠)
        session_id: UUID de sesi√≥n (para agrupar)
        request: HttpRequest (para extraer IP y user_agent)
    
    Returns:
        ActivityEvent: El evento creado
    """
```
